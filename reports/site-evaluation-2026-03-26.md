# サイト評価（2026-03-26）

## 総評
- 学習導線（トップ→学習/Q&A/更新）と最低限の品質ゲート（内部リンク/画像チェック）が整っており、継続改善しやすい構成。
- 一方で、ページ単位の実装揺れ（レイアウト指定やMarkdown記法）によって、表示崩れや更新ミスが起きやすい箇所が残っている。
- 直近は「テンプレートの統一」と「測定設定（Analytics/feedback URL実運用）」を優先すると、運用効率と改善速度が上がる。

## 良い点
1. **CIで内部品質を担保**
   - `_site` を対象に、0バイト画像・内部アセット欠損・内部リンク切れを検知できる。
2. **情報設計が明確**
   - ナビゲーションは学習・Q&A・過去問・更新など、実務者の目的起点で整理されている。
3. **Q&A投稿導線が複線化**
   - GitHub Issue と Googleフォームの2系統があり、参加ハードルが低い。

## 課題（優先度順）
1. **P1: ページ実装の一貫性不足**
   - レイアウトや記法の揺れがあると、表示異常や将来の保守コスト増につながる。
2. **P1: 計測設定が未投入**
   - `_config.yml` の analytics 設定が空で、改善判断の定量根拠が不足しやすい。
3. **P2: フィードバック導線の計測連携不足**
   - フォームへの導線はあるが、記事単位の改善ループ（回収→分析→反映）を回す設計が弱い。

## すぐ効く改善アクション（1〜2週間）
- `layout` 指定を全ページで棚卸しし、存在しないレイアウト名を除去。
- 主要ページ（home / learn / qa / guides）の「最終更新日」「次の行動CTA」を統一。
- GA4 か Matomo のどちらか1つを有効化し、検索語・遷移・離脱を見える化。
- Q&A投稿完了後のサンクス導線に「関連Q&A/解説」リンクを追加し、回遊率を改善。
