# github-pr-writer（変更提案→差分→PR文章スキル）

## 目的
- 実装の差分（diff）と、PR説明/リリースノートまで自動化

## 入力
- 変更前後ファイル
- 目的

## 実行手順
1. 変更の背景と目的を1段落で要約
2. 差分を機能単位で整理（追加/変更/削除）
3. 影響範囲・検証手順・既知リスクを明示
4. 失敗時のロールバック手順を作る
5. PRタイトルと本文をレビューしやすい形に整える

## 出力
1. diff案
2. PR本文
3. 検証手順
4. ロールバック手順

## ルール
- レビュアーが最短で判断できる順番で記述する
- 手順はコピペ実行可能な粒度で書く
