# info-architecture（情報設計スキル）

## 目的
- カテゴリ設計・ナビ構造・回遊導線を整える

## 入力
- 現行メニュー
- 記事一覧
- 目的（集客 / 信頼 / 収益）

## 実行手順
1. 現行メニューの重複・不足・優先順位のズレを抽出する
2. 目的に合わせてトップ導線を3本まで定義する
3. カテゴリを「主カテゴリ」「補助カテゴリ」に再編する
4. ヘッダー / フッター / 記事末尾の回遊導線を設計する
5. モバイルでタップしやすいナビ順を確定する

## 出力
1. 新メニュー案（ヘッダー/フッター）
2. カテゴリ定義（各カテゴリの目的・含める記事）
3. 配置図（文章でOK）

## ルール
- 迷ったら「ユーザーが次に何を押すか」が明確な案を優先する
- 1画面にCTAを詰め込みすぎず、主導線を明確化する
