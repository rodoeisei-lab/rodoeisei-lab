# rapid-improver（スピード優先改善スキル）

## 目的
- 完璧より前進を優先し、短時間で改善を実装する
- 事前確認なしで、効果が高い変更をまとめて実施する
- 変更点と再現手順を最小コストで共有する

## 入力
- 改善したい対象（機能、ページ、運用）
- 期待する効果（例: 作業時間短縮、可読性向上、導線改善）
- 既存コード/ファイル一式

入力が曖昧でも停止せず、妥当な前提を置いて進める。

## 実行手順
1. **最小探索で着手**
   - 最初は関連ファイルだけを読む（目安10ファイル）
   - 詰まったら探索範囲を拡大する
2. **高インパクト変更を先にまとめる**
   - 小粒改善を分散せず、整合する範囲で一括反映する
   - 無関係な修正は避けるが、整合のための同梱は許容する
3. **確認コストを下げる**
   - 事前承認・過剰な質問を省略する
   - 仕様不明点は合理的に仮定し、後段で明記する
4. **最小限の検証だけ行う**
   - 原則として build/test/lint は省略
   - 必要時のみ代表的な確認を実行して結果を短く残す
5. **共有しやすい形で出力する**
   - 3〜6行の要約
   - 変更ファイル一覧
   - 重要差分の箇条書き
   - 最小の再現手順

## 出力フォーマット
1) 何を変えたか（要約: 3〜6行）
2) 変更したファイル一覧
3) 重要な差分（要点: 箇条書き）
4) 動作確認手順（再現手順: 最小）

## ルール
- 迷ったら「速く前進できる案」を優先する
- 破壊的変更は避けないが、影響点は簡潔に明示する
- 説明は短く、意思決定に必要な情報だけを残す
